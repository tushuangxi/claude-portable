Claude Code Portable

便携版 Claude Code + CC Switch · 零安装，即插即用

快速开始

🍎 macOS双击 ClaudePortable.command
🐧 Linuxchmod +x ClaudePortable.sh && ./ClaudePortable.sh
🪟 Windows双击 ClaudePortable.bat

首次运行会自动引导你配置 API。推荐选择 CC Switch 图形界面（选项 1），添加 Provider 后关闭窗口即可。

工作原理

启动脚本做了三件事：

  1. 启动 CC Switch 代理 — 在本地监听端口，转发 API 请求到你配置的 Provider
  2. 设置环境变量 — 让 Claude Code 通过本地代理连接
  3. 启动 Claude Code — 进入交互式 CLI
CC Switch 支持多 Provider 切换、负载均衡、故障转移。你可以随时打开 CC Switch 修改配置。

配置 API

方式一：CC Switch GUI（推荐）

首次运行选择 1，或直接运行 bin/ 目录下的 cc-switch 可执行文件。在图形界面中添加你的 API Provider：

  • 填写 Base URL（如 https://api.anthropic.com 或第三方兼容端点）
  • 填写 API Key
  • 保存后关闭 CC Switch

方式二：命令行快速配置

首次运行选择 2，按提示输入 Base URL 和 API Key。配置保存在 config/ccswitch/providers.json。

环境变量

变量说明
ANTHROPIC_BASE_URLAPI 端点（代理模式自动设置）
ANTHROPIC_API_KEYAPI Key（代理模式自动设置）
CLAUDE_CONFIG_DIRClaude 配置目录（指向 data/.claude）
CLAUDE_HOMEClaude 主目录（指向 data/.claude）

便携性

所有数据存储在 data/ 目录内，不污染系统 HOME 目录。你可以：

  • 把整个文件夹复制到 U 盘，在不同电脑间使用
  • CC Switch 配置会自动在便携包和系统之间同步
  • 会话历史跟随便携包移动

常见问题

macOS 提示"无法打开，因为无法验证开发者"

启动脚本会自动移除 quarantine 属性。如果仍然被拦截，手动执行：

xattr -dr com.apple.quarantine /path/to/ClaudeCodePortable

CC Switch 代理未就绪

脚本会自动回退到直连模式，从 DB 或 providers.json 读取配置直接连接 API。功能不受影响。

更新 Claude Code

替换 bin/ 目录下对应平台的二进制文件即可。配置和数据不受影响。

Windows 上首次运行检测不到配置

确保使用 Release 页面下载的 zip 包（包含 sqlite3.exe）。如果是从源码运行，脚本会通过文件大小检测兜底。
