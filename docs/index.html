第一次使用：四步上手

如果你是第一次运行，按下面顺序操作就能用了。整个过程不到 2 分钟。

1
解压到电脑或 U 盘

把下载的 zip 文件解压到一个方便访问的位置。如果想做成"插哪用哪"的便携包，直接解压到 U 盘根目录。

注意：U 盘格式必须是 NTFS 或 exFAT，不支持 FAT32。

2
双击启动文件

找到对应你系统的启动文件，双击它：

  • Windows：双击 ClaudePortable.bat
  • macOS：双击 ClaudePortable.command
  • Linux：双击 ClaudePortable.sh 或在终端运行

启动后会出现一个黑色终端窗口和 YLYG logo，这是正常的。

3
在 CC Switch 里添加 Provider

首次运行时脚本会自动弹出 CC Switch 图形界面（一个独立的窗口）。在里面：

  • 点击 "添加 Provider" 或 "+" 按钮
  • 填写你的 API Base URL（如 https://api.anthropic.com 或第三方端点）
  • 填写你的 API Key
  • 选择 类型为 Claude
  • 点击 保存（不需要关闭 CC Switch 窗口）

脚本会自动检测到你保存了配置，几秒后继续。

4
确认信任目录，开始对话

Claude Code 启动后会问 "Is this a project you trust?"，按 Enter 选择 "Yes, I trust this folder"。

看到 > 提示符就可以开始对话了。试着输入 "你好" 测试一下。

★
之后再用就简单了：双击启动文件 → 直接进入 Claude Code 对话界面，不再需要任何配置。

关于防复制保护

首次成功配置后，便携包会自动绑定到当前 U 盘或硬盘。这意味着：

  • ✅ 你可以把 U 盘插到不同电脑，正常使用配置跟随
  • ❌ 别人复制你的 ClaudeCodePortable 文件夹到他们的电脑，会被拒绝运行
  • 🔓 如果你自己想换新 U 盘，运行 ClaudePortable.bat --unlock（macOS/Linux 同名）解绑后再迁移
⚠
重要：如果格式化 U 盘后重新解压，绑定会自动重新生成（无影响）。但如果想保留配置，请先备份 data/.cc-switch/ 目录。

切换 Provider

已经配置好之后，想换 API 提供商或切换不同 Key？打开 bin/ 目录下的 cc-switch（macOS 是 .app，Windows 是 cc-switch.exe），在 GUI 里：

  • 添加新的 Provider
  • 点击对应 Provider 的"激活"按钮切换
  • 保存后，下次启动 Claude Code 自动使用新配置

目录结构

ClaudeCodePortable/
├── bin/                    # 各平台二进制
│   ├── macos-arm64/        # Apple Silicon
│   ├── macos-x64/          # Intel Mac
│   ├── linux-x64/          # Linux x86_64
│   └── windows-x64/        # Windows 64-bit
├── lib/                    # PowerShell / shell 辅助脚本
├── data/                   # 你的所有数据（首次运行后生成）
│   ├── .cc-switch/         # CC Switch 数据（API Keys、provider 配置）
│   ├── .claude/            # Claude Code 会话历史
│   ├── .lock               # 防复制绑定锁
│   └── .running            # 单实例运行锁
├── ClaudePortable.command  # macOS 启动脚本
├── ClaudePortable.sh       # Linux 启动脚本
└── ClaudePortable.bat      # Windows 启动脚本

常见问题

启动后说 "No provider found"

这意味着 CC Switch 还没有保存有效的 Provider 配置。返回 CC Switch 窗口，确认你填写了 Base URL 和 API Key 并点击了保存按钮。

提示 "locked to its original USB drive"

说明这个文件夹是从别处复制过来的（防复制保护生效）。如果是你本人想迁移：

ClaudePortable.bat --unlock        # Windows
./ClaudePortable.command --unlock  # macOS
./ClaudePortable.sh --unlock       # Linux

macOS 提示 "无法验证开发者"

启动脚本会自动处理。如果仍然被拦截，在终端里执行：

xattr -dr com.apple.quarantine /path/to/ClaudeCodePortable

Windows 上首次运行报错或卡住

确保：

  • U 盘格式是 NTFS 或 exFAT（不能是 FAT32）
  • 没有杀毒软件拦截 mklink 命令
  • 解压路径不要太深，不要含中文（C:\ClaudePortable\ 是好的位置）

"Auth conflict" 警告

如果之前自己装过 Claude Code，环境变量可能冲突。脚本启动时会自动清理。如果还有问题，关闭所有终端重新双击启动文件。

更新到新版本

下载新版 zip，解压到一个新文件夹，把旧文件夹的 data/ 目录复制过去覆盖。配置、对话历史、绑定锁全部保留。

双击没反应（Linux）

大部分 Linux 文件管理器默认不会双击执行 .sh。打开终端：

cd /path/to/ClaudeCodePortable
chmod +x ClaudePortable.sh
./ClaudePortable.sh
