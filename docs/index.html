第一次使用：四步上手

整个过程不到 2 分钟。

1
把 U 盘插到电脑

插好后打开 U 盘，找到 ClaudeCodePortable 文件夹，进入里面。

2
双击启动文件

找到对应你系统的启动文件，双击它：

  • Windows：双击 ClaudePortable.bat
  • macOS：双击 ClaudePortable.command
  • Linux：双击 ClaudePortable.sh 或在终端运行

启动后会出现一个黑色终端窗口和 YLYG logo，这是正常的。

第一次启动会自动完成绑定（绑定到你这块 U 盘），需要等几秒。

3
浏览器自动打开"配置中心"（如果是首次配置）

如果 U 盘是全新未配置的，会自动打开浏览器进入配置中心（地址 http://127.0.0.1:17580）。跟着向导四步走：

  • 选供应商：从 13 个内置供应商里挑一个（Anthropic 官方、DeepSeek、智谱 GLM、Kimi、通义千问、MiniMax、豆包、Gemini、Grok、Groq、OpenRouter、硅基流动，或自定义中转站）
  • 填 Key：粘贴 API Key，需要时选一个模型（如 claude-opus-4-8、deepseek-v4-pro）
  • 测试：点"开始测试"确认 Key 和端点可用
  • 保存：保存后关闭浏览器页面，回到终端即可

如果卖家已经在 U 盘里预配置好了，这一步会自动跳过，直接进入对话。

4
确认信任目录，开始对话

Claude Code 启动后会问 "Is this a project you trust?"，按 Enter 选择 "Yes, I trust this folder"。

看到 > 提示符就可以开始对话了。试着输入 "你好" 测试一下。

★
之后再用就更简单：插 U 盘 → 双击启动文件 → 直接进入 Claude Code 对话界面。
⚡
零安装承诺：整个便携包不依赖宿主机环境，不需要你装 Python、Node 或任何运行时。配置中心用的运行时已经打包在 U 盘里，开箱即用。

关于 U 盘绑定

这块 U 盘已经为你配置好了 API 和绑定。你可以把 U 盘插到任何电脑使用，配置都跟着 U 盘走。但出于安全考虑，文件夹被复制到其他设备运行时会被拒绝。

切换大模型 / Provider

随时想换供应商或模型，不用重新跑首次配置，直接打开配置中心：

Windows 命令行进入 U 盘目录后运行
ClaudePortable.bat --config
macOS 终端里运行
./ClaudePortable.command --config
Linux 终端里运行
./ClaudePortable.sh --config

浏览器会打开配置中心。在里面你可以：

  • 模型标签：改当前供应商、Key、模型，点"保存配置"即生效
  • 供应商标签：在已保存的多个供应商之间一键切换
  • 运维标签：诊断环境、查看配置（脱敏）、导出/导入配置、查看日志、解绑设备

下次双击启动文件时，Claude Code 会自动使用你当前激活的供应商。不需要重新配置任何东西。

💡
小贴士：配置中心可以同时保存多个供应商，比如 Anthropic 官方、DeepSeek、智谱 GLM 等。随时一键切换，不用重新输 API Key。它和 CC Switch 共用同一份配置库，两边的修改互相同步。

想用 CC Switch 原生界面？

配置中心已经能完成日常的供应商管理。如果你想用 CC Switch 的原生图形界面（带故障转移、多应用切换等高级功能），打开配置中心 → 运维标签 → 点 "启动 CC Switch" 按钮即可。两边共用同一份配置库，改哪边都同步。

常见问题

启动后说 "No provider found" 或没检测到配置

这种情况很少出现。如果遇到，先重新插拔 U 盘再双击启动文件。若仍有问题，用 --config 参数手动打开配置中心重新保存一次。

配置了 DeepSeek 但 Claude 界面显示 Opus 4.8？

这是旧版本的已知问题，已在最新版修复。请确认你的便携包是 v0.6.0 或更新版本。新版启动器会正确读取并传递你在配置中心选择的模型。

macOS 提示 "无法验证开发者" 或 "无权限执行"

第一次双击 .command 文件可能提示无法执行。两种处理方式：

  • 右键打开（推荐）：右键点击 ClaudePortable.command → 选择"打开" → 在弹窗中再点"打开"。下次就能双击运行了。
  • 用终端：打开"终端"，输入 chmod +x （带空格），把 ClaudePortable.command 文件拖到终端里，回车即可。同样处理一遍 lib/binding.sh。

这是 macOS 自带解压工具的限制 — 解压 zip 时会丢失文件的执行权限。用 The Unarchiver 或 Keka 解压可以避免此问题。

Windows 上首次运行报错或卡住

常见原因：

  • U 盘格式问题：联系卖家更换
  • 杀毒软件拦截 mklink 命令：临时关闭杀毒软件

"Auth conflict" 警告

如果之前自己装过 Claude Code，环境变量可能冲突。脚本启动时会自动清理。如果还有问题，关闭所有终端重新双击启动文件。

双击没反应（Linux）

大部分 Linux 文件管理器默认不会双击执行 .sh。打开终端：

cd /path/to/ClaudeCodePortable
chmod +x ClaudePortable.sh
./ClaudePortable.sh
